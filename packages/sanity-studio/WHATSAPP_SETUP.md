# 📱 تكامل واتساب للأعمال — عيادة إيات

## ✅ ما تم إنشاؤه

### 1. Schema Types جديدة في Sanity

#### `whatsappTemplate` — قوالب الرسائل
- **name**: اسم القالب الداخلي
- **category**: نوع القالب (عروض / باقات / مواعيد / متابعة / مخصص)
- **messageBody**: نص الرسالة مع دعم المتغيرات:
  - `{{اسم_المريض}}`
  - `{{اسم_العرض}}`
  - `{{السعر}}`
  - `{{تاريخ_الموعد}}`
  - `{{اسم_الطبيب}}`
- **linkedOffer**: ربط بعرض موجود (يملأ البيانات تلقائياً)
- **linkedService**: ربط بباقة علاج
- **includeCallButton**: زر اتصال في نهاية الرسالة
- **active**: تفعيل/إيقاف القالب

#### `whatsappConversation` — سجل المحادثات
- سجل قراءة فقط لتتبع الرسائل المرسلة

### 2. Custom Tool في Sanity Dashboard
أداة `واتساب` مضافة في شريط الأدوات العلوي للـ Studio تتضمن:
- **قائمة القوالب** مع فلتر حسب النوع
- **نموذج الإرسال** مع المتغيرات الديناميكية
- **معاينة الرسالة** قبل الإرسال
- **سجل الإرسال** للجلسة الحالية
- **إحصائيات** سريعة

## 🚀 كيفية الاستخدام

### إضافة قالب جديد
1. افتح Sanity Studio
2. من القائمة الجانبية: **📱 واتساب للأعمال** → **📋 قوالب الرسائل**
3. اضغط **Create new**
4. أدخل:
   - اسم القالب
   - نوعه (عرض / باقة / موعد...)
   - نص الرسالة (يمكن استخدام المتغيرات)
   - اربطه بعرض أو باقة موجودة (اختياري)
5. احفظ

### إرسال رسالة
1. اضغط على أيقونة **📱 واتساب** في شريط الأدوات العلوي
2. اختر قالباً أو اكتب رسالة مخصصة
3. أدخل اسم المريض ورقم الواتساب
4. اضغط **📤 إرسال عبر واتساب**

## 📝 إضافة قوالب افتراضية

لإضافة القوالب الافتراضية تلقائياً، تحتاج إلى:

1. اذهب إلى [Sanity Manage](https://sanity.io/manage/personal/project/f46widyg/api#tokens)
2. أنشئ **Editor Token** جديد
3. استبدل الـ Token في ملف `seed-whatsapp-templates.mjs`
4. شغّل: `node seed-whatsapp-templates.mjs`

## 🔒 ملاحظة أمان

الـ Access Token لـ WhatsApp موجود في ملف `WhatsAppTool.tsx`. 
في بيئة الإنتاج يُنصح بـ:
1. نقل الـ Token إلى Sanity Secrets
2. أو إنشاء API proxy في eiat-site لإخفاء الـ Token
